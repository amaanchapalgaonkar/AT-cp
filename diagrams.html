🔍 NFA vs DFA Comparison

NFA (Thompson's Construction)

  • Multiple transitions per symbol
  • ε-transitions (epsilon moves)
  • Smaller state count
  • Harder to simulate directly
  • Used as intermediate step

DFA (Subset Construction)

  • Single transition per symbol
  • No ε-transitions
  • Larger state count (subset of NFA states)
  • Fast to simulate
  • Used for actual recognition

Minimized DFA (Hopcroft)

  • Optimal state count
  • Fastest recognition
  • Unique for each language
  • Equivalent to original DFA
  • Used in production

ε-Closure Example

When converting NFA to DFA, we use ε-closure to group states reachable via ε-transitions:

NFA with ε-transitions
graph LR A((q0)):::start -->|"a"| B((q1)) B -->|"eps"| C((q2)) C -->|"eps"| D((q3)):::accepting A -->|"eps"| E((q4))

ε-closure(q0) = {q0, q4}
ε-closure(q1) = {q1}
ε-closure(q2) = {q2, q3}
ε-closure({q2,q3}) = {q2, q3}

Resulting DFA States
graph LR A[["q0q4"]]:::start -->|"a"| B[["q1"]] B -->|"eps"| C[["q2q3"]]:::accepting

DFA states = powerset of NFA states
Accepting states include any ε-closure containing accepting NFA state

⚡ Longest Match (Maximal Munch) Strategy

When multiple tokens match at a position, the tokenizer chooses the longest one. This resolves conflicts and ensures optimal tokenization.

Example Input:
john@gmail.com is an email
Position Possible Matches Chosen Reason
0-4 WORD: "john" WORD: "john" Only match
4-17 MENTION: "@gmail", EMAIL: "@gmail.com" (if supported) Longest match Consumes more chars
17-20 WORD: "is" WORD: "is" Only match
20-26 WORD: "an", "a" WORD: "an" Longest from position
